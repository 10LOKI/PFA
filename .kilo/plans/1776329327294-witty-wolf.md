# Plan: Partner Reward Creation Feature

## Objectif
Permettre aux partenaires de créer des récompenses que les étudiants peuvent acheter avec leurs points.

## État Actuel
- Le système de récompenses existe déjà (`Reward` model)
- Les étudiants peuvent échanger leurs points contre des récompenses (`RedeemRewardAction`)
- **AUCUNE** interface n'existe pour que les partenaires créent des récompenses

## Structure Existante
- `Reward` model: `partner_id`, `title`, `description`, `image`, `points_cost`, `stock`, `min_grade`, `is_premium`, `is_active`, `expires_at`
- `RewardController`: uniquement `index` et `redeem` (pas de création)
- Routes: pas de routes pour créer des récompenses

## Travaux à Réaliser

### 1. Routes (routes/web.php)
Ajouter:
- `GET /rewards/create` → RewardController@create (formulaire création)
- `POST /rewards` → RewardController@store (enregistrer)
- `GET /rewards/{reward}/edit` → RewardController@edit (formulaire modification)
- `PUT /rewards/{reward}` → RewardController@update (mettre à jour)
- `DELETE /rewards/{reward}` → RewardController@destroy (supprimer)

### 2. RewardController
Ajouter les méthodes:
- `create()` - retourne vue avec formulaire
- `store(Request $request)` - crée une nouvelle récompense
- `edit()` - retourne vue avec formulaire modification
- `update(Request $request, Reward $reward)` - met à jour
- `destroy(Reward $reward)` - supprime

### 3. Vues
- `resources/views/rewards/create.blade.php` - formulaire création
- `resources/views/rewards/edit.blade.php` - formulaire modification

### 4. Policy
Créer `RewardPolicy` avec:
- `create()` - seul un partenaire peut créer
- `update()` - seul le partenaire propriétaire peut modifier
- `delete()` - seul le partenaire propriétaire peut supprimer

## Considérations
- Le `partner_id` sera automatiquement défini depuis `auth()->user()`
- Validation: title requis, points_cost > 0, stock >= 0 (ou null pour illimité)
- Grade requirements: novice/pilier/ambassadeur
- Image upload optionnel (stocker dans storage/app/public/rewards)
