# Plan: Check-Out System (Option A)

## Objectif
Implémenter un système de check-out obligatoire pour valider la durée réelle de présence et prévenir le gaming du système par scan-and-run.

## Problème Résolu
- Étudiant scanne QR, part 15min après, obtient 4h de présence payées
- solution: `checked_out_at` dans event_user, validation durée réelle

## Schéma Actuel vs Cible

### Migration: event_user (ajouter checked_out_at)
```php
$table->timestamp('checked_out_at')->nullable();  // AJOUTER
```

### Modèle: EventUser (modifier)
- Ajouter `hasCheckedOut()` scope
- Ajouter `actualDurationInMinutes()` accessor
- Ajouter `proRatedHours()` accessor

### Action: CheckOutStudentAction (CRÉER)
- Called after CheckInStudentAction OR standalone (student forgot to check-in)
- Validates: `checked_in_at` must be set
- Enforces: `checked_out_at` = NOW()
- Calculates: `hours_earned = MIN(actual_duration, event.duration_hours)`
- Calls: `CreditPointsAction` with pro-rated amount

### Route: POST /events/{event}/checkout (CRÉER)
- CheckOutController@store
- Authorization: student + registered + hasCheckedIn() + no checked_out_at

### PointsTransaction: amount calculation change
- Change from: `event.duration_hours * event.points_reward`
- To: `proRatedHours() * event.points_reward`

## Implémentation

### 1. Migration
- `checked_out_at` nullable timestamp dans event_user

### 2. Modèle EventUser
- `hasCheckedIn(): bool` (EXISTE)
- `hasCheckedOut(): bool` - NEW
- `actualDurationInMinutes(): int` - NEW
- `proRatedHours(): float` - NEW

### 3. CheckOutStudentAction
- Validate: already checked_in
- Validate: not already checked_out
- Set: checked_out_at = now()
- Calculate: hours_earned = min(actual_duration, duration_hours)
- Credit: points with pro-rated amount
- Update: pivot points_earned

### 4. Routes
- POST /events/{event}/checkout → CheckOutController@store
- GET /events/{event}/qr → add checkout option? NO - partner must validate

### 5. Frontend
- Dashboard student: show "Check Out" button for checked-in events
- OR Partner validates with second scan (better UX)

## Business Logic Règles
1. `actual_duration < 30min` → 0 points (no-show penalty)
2. `actual_duration >= event.duration_hours` → full points
3. `30min <= actual_duration < duration_hours` → pro-rated points
4. Never credit more than event.duration_hours
