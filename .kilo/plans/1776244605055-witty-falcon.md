# Plan: DM System Enhancement - User Discovery

## Objective
Enable users to discover and contact other users via Direct Messages:
1. **Event Participants List** - Collapsible view of users registered in an event + message button
2. **Global User Search** - Search and contact any user via search bar
3. **Real-time Messaging** - Instant message delivery using Laravel Broadcasting + Pusher

## Current State
- Tables: `conversations` (basic), `messages` (Model exists but migration missing), `conversation_user` (pivot - needs migration)
- Models: `Conversation`, `Message`, User relationships
- Controller: `MessageController` with index, show, store, start
- Views: `messages/index.blade.php`, `messages/show.blade.php`
- Routes: `/messages`, `/messages/{conversation}`, `/messages/start`

## Implementation Plan

### Phase 1: Database Fixes
1. Create `messages` table migration:
   - id, conversation_id, sender_id, body, read_at, timestamps
2. Create `conversation_user` pivot table migration:
   - conversation_id, user_id, timestamps
3. Run migrations: `php artisan migrate`

### Phase 2: Real-Time Messaging Setup
1. Install Pusher: `composer require pusher/pusher-php-server`
2. User needs to add these to `.env`:
   ```
   PUSHER_APP_ID=your_app_id
   PUSHER_KEY=your_key
   PUSHER_SECRET=your_secret
   PUSHER_CLUSTER=your_cluster
   BROADCAST_DRIVER=pusher
   ```
3. Update `config/broadcasting.php` with Pusher driver (or verify it's already set)
4. Create `MessageSent` event class:
   - Implements `ShouldBroadcast`
   - Broadcasts to `conversation.{id}` channel
5. Update `MessageController@store` to dispatch `MessageSent` event

### Phase 3: User Search & Discovery
1. Create `UserController` with:
   - `index` - list all users with search (name, email)
   - Search: `User::where('name', 'like', "%{$q}%")->orWhere('email', 'like', "%{$q}%")`
   - Exclude: current user from results
2. Add route: `GET /users` (requires auth)
3. Create `users/index.blade.php`:
   - Search bar at top
   - Grid of user cards with avatar, name, role, "Message" button

### Phase 4: Event Participants Integration
1. Update `events/show.blade.php`:
   - Add collapsible "Participants" section (using `<details>` or AlpineJS)
   - List registered users with "Message" button
   - Button triggers: `POST /messages/start` with `user_id`

### Phase 5: Message UI Enhancements
1. Update `messages/index.blade.php`:
   - Add "New Message" button → goes to `/users` page
   - Search conversations
2. Update `messages/show.blade.php`:
   - Add real-time JavaScript listener for new messages
   - Auto-scroll to bottom on new message
