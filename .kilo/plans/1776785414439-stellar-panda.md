# Plan: Student-Specific QR Codes for Event Check-in

## Current State
- Events have ONE `qr_code_token` stored in `events` table
- Partners display QR, manually type token + select student from dropdown
- Two-step manual check-in process

## Target State
- Each student registration gets its OWN unique QR code
- QR encodes both event ID and user ID (bound to specific student)
- Partner scans student's individual QR → automatic check-in
- No manual token entry or student selection needed

---

## Implementation Steps

### Step 1: Database Migration
**File**: `database/migrations/YYYY_MM_DD_HHMMSS_add_qr_token_to_event_user_table.php`
- Add `qr_token` column to `event_user` pivot table (string, unique, nullable)
- This token uniquely identifies the specific registration

### Step 2: Model Updates
**File**: `app/Models/EventUser.php`
- Add `qr_token` to `$fillable`
- Generate QR token automatically when registration is created (in booted method or via observer)

### Step 3: New QR Generation Action
**File**: `app\Actions\Event\GenerateStudentQrAction.php` (NEW)
- Generates a unique token (UUID) for a specific EventUser pivot record
- Creates QR code SVG using bacon/bacon-qr-code library
- QR code content: The unique token (or a URL like `/checkin/{token}`)

### Step 4: New Check-in Scanning Route & Controller
**File**: `app\Http\Controllers/StudentCheckInController.php` (NEW)
- Single route that handles scanning: `GET /checkin/{token}` or `POST /checkin/scan`
- Validates token exists in `event_user.qr_token`
- Verifies event is valid (approved, within time window?)
- Checks student hasn't already checked in
- Performs check-in (updates pivot: status, checked_in_at)
- Shows success/failure page

**Alternative approach**: Keep POST with CSRF protection, use a simple token field like current system but token is student-specific

### Step 5: Update Event Registration & Email QR
**Files**: 
- `app/Http/Controllers/EventRegistrationController.php`
- `app/Mail/StudentEventQrMail.php` (NEW)

On successful registration:
- Generate unique `qr_token` for pivot (via EventUser model)
- Send email to student with QR code (using Mailtrap)
- Email includes event details and QR SVG embedded

**Note**: Students can only register if event is approved (already enforced).

### Step 6: Student QR Display on Event Page
**File**: `resources/views/events/show.blade.php`
Add section for registered students:
- Show "YOUR CHECK-IN QR" panel with generated QR code
- QR visible only to the student who registered (auth check)
- Link to full "My QR Codes" page

**Optional**: Create `StudentQrController` with `myQrCodes()` method for listing all.

### Step 7: New QR Generation Action
**File**: `app\Actions\Event\GenerateStudentQrAction.php` (NEW)
- Public function `execute(EventUser $registration): string`
- If registration has no `qr_token`, generate `Str::uuid()->toString()` and save
- Create QR SVG encoding URL: `route('checkin.scan', $token)` (GET route)
- Return SVG string

Used by: email, event page display, manual generation if needed.

### Step 8: New Check-in Scanning Route & Controller
**Files**: 
- `app\Http\Controllers\StudentCheckInController.php` (NEW)
- Route additions in `routes/web.php`:
  - `GET|POST /checkin/{token}` (name: `checkin.scan`) - partner scanning endpoint

**Logic**:
1. Auth middleware (partner only)
2. Find `EventUser` by `qr_token` = $token (404 if not found)
3. Validate:
   - Event is approved
   - Event end time >= now() (check-in window)
   - Student not already checked in
4. Update pivot: `status='checked_in'`, `checked_in_at=now()`
5. Redirect back to event page with success/fail message

**Alternative**: Keep old POST `/events/{event}/checkin` for fallback manual (partner enters student+token separately). The new single-token endpoint replaces it.

### Step 9: Partner Check-in Interface Rebuild
**File**: `resources/views/events/show.blade.php` - Check-in Terminal section
Replace two-field form with:
- **QR Scanner input** (HTML5): `<input type="text" id="qr-input" placeholder="Scan QR or paste token">` with JS to capture camera via `navigator.mediaDevices.getUserMedia()` or use file upload button that reads QR image and extracts text (using `html5-qrcode` library). Simpler: Just a text field and paste instructions.
- **Manual fallback**: Token input field with "SUBMIT CHECK-IN" button that goes to `/checkin/{token}`.

Remove the student dropdown selection entirely.

**JavaScript inclusion** (optional): CDN link for `html5-qrcode` scanner library. Provide both scan and manual entry.

### Step 10: Check-out Policy
Keep manual partner-validated check-out (current system). No changes needed.

### Step 11: Remove Old QR System
**Migrate & Remove**:
- Create migration: drop column `qr_code_token` from `events`
- Delete file: `app/Actions/Event/GenerateQrAction.php`
- Remove method `qr()` from `app/Http/Controllers/EventController.php`
- Remove route `events.qr` (`GET events/{event}/qr`) from `routes/web.php`
- Remove `qr_code_token` from `Event` model `$fillable`
- Remove any `event.generate-qr` permission assignments from seeders (if present)

### Step 12: Model Boot & Token Generation
**File**: `app/Models/EventUser.php`
Add in `booted()` method:
```php
static::creating(function ($registration) {
    if (empty($registration->qr_token)) {
        $registration->qr_token = Str::uuid()->toString();
    }
});
```
Add `use Illuminate\Support\Str;`.

### Step 13: Testing
Write Pest tests for:
- `EventUser` creation auto-generates unique `qr_token`
- `GenerateStudentQrAction` returns valid SVG containing token URL
- `StudentCheckInController` success flow (valid token)
- `StudentCheckInController` rejects: invalid token, expired event, already checked-in
- Email sent on registration contains QR

### Step 14: Assets & Frontend
- Include optional QR scanner library (e.g., `html5-qrcode` via CDN) in `events/show.blade.php` for camera scanning.
- Style check-in terminal to look like terminal UI (already matches theme).
