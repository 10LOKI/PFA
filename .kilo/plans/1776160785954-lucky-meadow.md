# Plan: UI & Landing Page ActTogether

## Contexte
L'utilisateur veut ajouter le UI pour toutes les pages + une landing page.
Le projet utilise Laravel Blade + Tailwind CSS v4.

## État Actuel des Views

### ✅ Views Existantes (Breeze + Custom)
- `auth/` - login, register, forgot-password, reset-password, verify-email, confirm-password
- `dashboard/` - student.blade.php, partner.blade.php, admin.blade.php
- `events/` - index, show, create, edit
- `rewards/` - index
- `profile/` - edit, partials (update-profile, update-password, delete-user)
- `layouts/` - app, guest, navigation
- `components/` - 13 composants (buttons, inputs, etc.)

### ❌ Views Manquantes / À Refaire
- **welcome.blade.php** - Laravel default → doit devenir Landing Page ActTogether

## Plan de Travail

### Phase 1: Landing Page (Priorité HAUTE)
**Fichier:** `resources/views/welcome.blade.php`

À créer:
- Hero section avec logo + tagline ActTogether
- Features: Points System, QR Check-in, Rewards Marketplace
- Call-to-action: Login / Register
- Footer avec liens

### Phase 2: Navigation & Layout (Priorité HAUTE)
**Fichier:** `resources/views/layouts/navigation.blade.php`

À améliorer:
- Menu dynamique selon rôle (student/partner/admin)
- Links: Events, Rewards, Profile, Logout

### Phase 3: Pages Dashboard (Priorité MOYENNE)
À améliorer:
