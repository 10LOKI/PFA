# Real-time Event Creation Notifications

Notify all users when a new event is created.

## Approach

Use Laravel's **Notification** system with Pusher for real-time delivery:

1. **Create EventCreatedNotification** - A notification class that broadcasts event details
2. **Dispatch notification in EventController** - After event is created, notify all users
3. **Create Pusher notification channel** - Use Pusher for real-time browser/mobile notifications
4. **Add notification badge in UI** - Show notification bell with unread count

## Implementation

### 1. Create notification class
```
php artisan make:notification EventCreatedNotification
```

In `EventCreatedNotification`:
- Implement `ShouldBroadcast` interface
- Use `OnBroadcastableEvents` to use Pusher
- Store notification in database for history

### 2. Update EventController::store()
- After `$event = Event::create($data);`, dispatch notification to all users

### 3. Frontend JavaScript
- Use Pusher client to listen for `private-notifications.{user_id}` channel
- Show toast/notification when event created
- Add notification bell icon in navigation with unread count

## Notifications Scope

- **Recipients**: All students (or all users except the event creator)
- **Channel**: `private-notifications.{user_id}`
- **Data sent**: event title, category, points, starts_at, link

## Files to modify

- `app/Notifications/EventCreatedNotification.php` (new)
- `app/Http/Controllers/EventController.php` (add dispatch)
- `resources/views/layouts/app.blade.php` or navigation (add notification bell + JS)
